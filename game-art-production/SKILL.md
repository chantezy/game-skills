# 游戏美术生产：AI辅助工作流

## 概述

美术工作流覆盖原画设计（三阶段）、场景设计、特效制作、动画制作、UI设计五个方向。核心流程：**引导用户提供设计方向和参考素材 → AI据此生成美术规范文档 → 基于规范产出美术资产**。所有工作流均包含"如果不满意"反馈循环：不满意 → 优化提示词 → 重新生成。

## 用户输入引导

### 必需输入

- **设计方向描述**：希望产出的内容风格、主题、氛围
- **参考图片**：风格参考图、配色参考、构图参考（用户上传或描述）

### 可选输入

- **参照物图片**：具体的形状、轮廓参考
- **特定关键词**：如"硬笔刷"、"边缘光"、"饱和度"等美术专业术语
- **禁用元素**：明确不希望出现的视觉元素
- **配色方案要求**：指定的颜色组合或色彩倾向
- **相同风格参考**：已有项目中风格一致的参考资产

### 输入不足时的处理

当用户提供的信息不足以产出高质量美术资产时，AI应：
1. 明确指出缺少哪些关键信息
2. 提供选项供用户选择（如"请选择美术风格：A. 写实 B. 卡通 C. 像素 D. 水墨"）
3. 给出默认建议方案供用户确认或调整

## 工作流一：美术原画（三阶段）

### 第一阶段：草图设计

**输入：**

| 输入项 | 必需 |
|--------|------|
| 参照物图片 | 是 |
| 设计方向要求 | 是 |
| 配色或体型相似的角色立绘参考 | 否 |

**AI处理：**
1. 分析用户提供的参考图片，提取风格特征
2. 基于设计方向生成美术规范文档
3. 据此生成多版概念草图

**产出：**
- 美术规范文档（首次产出时）
- 多版有设计想法的概念草图
- 优化后的AI提示词
- 设计方向验证报告
- 创意方案评估

**反馈：** 如果不满意，反哺回用户输入，用户补充方向后重新生成

**提示词参考示例：**
"保持左右两边的风格和精细度一致，适当修整杂边和图形流畅度。参考第二张图的素描关系与光影表现，二分与高光，明暗交界线以及不同颜色的材质且都使用硬笔刷和硬边缘，不要软笔刷，边缘光和反光以及色彩的饱和度。细化第一张图且不改变外轮廓，把第一张图和第二张图相同配色的部分做相同的美术材质表现和细节处理，背景是白色，线稿颜色要淡一些。"

### 第二阶段：原画细化

**输入：**

| 输入项 | 必需 |
|--------|------|
| 选定草图方案（从第一阶段产出中选择） | 是 |
| 特定细化关键词 | 否 |
| 相同风格角色参考 | 否 |
| 配色方案要求 | 否 |

**AI处理：**
1. 基于美术规范文档和选定草图进行细化
2. 检查风格一致性
3. 评估细节完善度

**产出：**
- 多版细化完成度高的原画
- 风格一致性检查报告
- 细节完善度评估
- 最终原画方案

### 第三阶段：部件拆分

**输入：**

| 输入项 | 必需 |
|--------|------|
| 最终原画方案（从第二阶段产出中选择） | 是 |
| 动画拆分逻辑关键词 | 否 |
| 部件功能要求 | 否 |
| 动画制作规范 | 否 |

**AI处理：**
1. 分析原画，识别可拆分的动画部件
2. 标注部件功能
3. 生成动画制作建议

**产出：**
- 多版适合动画的部件拆分图
- 有效部件选取方案
- 部件功能标注
- 动画制作建议

## 工作流二：场景设计

**输入：**

| 输入项 | 必需 |
|--------|------|
| 前期快速手绘布局草图（可以是简笔画） | 是 |
| 场景设计要求 | 是 |
| 局部重绘需求 | 否 |

**AI处理：**
1. 分析手绘草图，理解布局意图
2. 生成场景规范
3. 产出多版场景草图

**产出：**
- 场景美术规范
- 多版有设计想法的场景草图
- 优化后的场景布局
- 局部重绘方案
- 场景设计验证报告

## 工作流三：特效制作

**输入：**

| 输入项 | 必需 |
|--------|------|
| 特定特效关键词（如"火焰爆发"、"冰冻扩散"） | 是 |
| 原画提供的对应素材垫图 | 是 |
| 特效表现要求（持续时间、强度、范围） | 否 |

**AI处理：**
1. 分析关键词和垫图，确定特效方向
2. 生成特效帧序列
3. 合成黑底特效视频

**产出：**
- 黑底特效视频
- 特效帧序列
- 特效表现评估
- 优化后的特效方案

## 工作流四：动画制作

**输入：**

| 输入项 | 必需 |
|--------|------|
| 分镜多宫图（定义动画节奏和关键帧） | 是 |
| 剧情动画要求（描述动画要表达的情绪和剧情） | 是 |
| 动画节奏要求（如"前慢后快"、"渐强"） | 否 |

**AI处理：**
1. 分析分镜，理解动画结构
2. 根据剧情要求生成动画
3. 评估节奏与表现力

**产出：**
- 完整剧情动画
- 动画节奏分析
- 剧情表现评估
- 动画优化建议

## 工作流五：UI设计

### 图标设计

**输入：** 特定关键词（必需）+ 相同风格和配色的图标参考（必需）
**产出：** 风格贴近的图标

### 字体标题设计

**输入：** 参考图相似风格的字体（必需）+ 想要的样式文字参考图（必需）
**产出：** 符合风格要求的字体标题设计

### 插图设计

**输入：** 特定关键词（必需）+ 根据角色立绘制造相对应元素的插图（必需）
**产出：** 拥有角色立绘的插图

### 界面设计

**输入：** 简单拼好的布局（必需，可以是线框图或手绘布局）
**产出：** 界面按钮边框跟角色元素统一性强的效果图

## AI提示词模板结构

### 一、角色立绘提示词模板（6模块结构）

**基础结构：** `[主体描述], [风格关键词], [配色要求], [细节要求], [背景要求], [质量关键词]`

| 模块 | 说明 | 示例 |
|------|------|------|
| 主体描述 | 角色身份、姿态、表情、服饰等核心特征 | "火属性角色战士，持剑站立姿态，身穿轻甲" |
| 风格关键词 | 美术风格、渲染方式、笔触特征 | "赛璐璐动漫风格，硬笔刷，清晰线稿" |
| 配色要求 | 主色调、辅助色、配色倾向 | "以红色和暗金为主色调，辅以白色高光" |
| 细节要求 | 材质表现、光影处理、装饰元素 | "金属反光质感，边缘光明显，服饰带有火焰纹路" |
| 背景要求 | 背景颜色或场景简述 | "纯白背景"或"透明背景" |
| 质量关键词 | 分辨率、完成度、精度要求 | "高精度，4K分辨率，细节丰富，完成度高" |

**完整示例 —— 赛璐璐动漫风格（Cel-shaded Anime）：**

```
火属性角色战士，年轻女性角色，持剑战斗姿态，身穿红黑配色轻甲，
赛璐璐动漫风格，硬笔刷，粗线稿，色块分明，无渐变过渡，
以红色(#CC2233)和暗金(#8B6914)为主色调，辅以白色高光点缀，
金属铠甲带有明显边缘光和反光，服饰边缘有火焰纹路装饰，头发飘逸带有光泽高光，
纯白背景，
高精度，2048×2048分辨率，细节丰富，完成度高，专业游戏立绘品质
```

**负面提示词模板（Negative Prompt）：**

```
低质量，模糊，变形，多余肢体，文字水印，签名，
过度饱和，过度曝光，线稿杂乱，比例失调，
anatomical errors, extra fingers, deformed face, blurry, low resolution,
watermark, signature, text, oversaturated, bad anatomy
```

### 二、场景背景提示词模板（5模块结构）

**基础结构：** `[场景类型], [时间段/光照], [氛围关键词], [构图要求], [风格关键词]`

| 模块 | 说明 | 示例 |
|------|------|------|
| 场景类型 | 场景的功能定位与环境类型 | "地下迷宫、森林村庄、火山洞穴" |
| 时间段/光照 | 时间设定和光源描述 | "黄昏时分，暖色夕阳从左侧照射" |
| 氛围关键词 | 情绪基调和环境感受 | "神秘、压迫感、宁静祥和" |
| 构图要求 | 透视方式、景深层次、焦点位置 | "一点透视，三层景深，焦点在中央建筑" |
| 风格关键词 | 美术风格和渲染特征 | "手绘厚涂，赛璐璐，像素风格" |

**完整示例 —— 地下迷宫场景：**

```
古代遗迹地下迷宫，宽阔的石质走廊，两侧有断裂的石柱和苔藓覆盖的墙壁，
夜晚/无自然光，场景由散落的魔法水晶提供冷蓝色点光源，远处有微弱的火把暖光，
神秘压抑氛围，带有探索未知感，寂静中隐含危险暗示，
一点透视构图，纵深感强，前景有碎裂的石块和藤蔓，中景为走廊主体，远景消失在黑暗中，
手绘厚涂风格，材质表现细腻，石质纹理清晰，光影对比强烈，
1920×1080分辨率，宽屏比例，游戏场景背景品质
```

### 三、UI图标提示词模板（4模块结构）

**基础结构：** `[图标内容], [风格], [配色], [背景]`

| 模块 | 说明 | 示例 |
|------|------|------|
| 图标内容 | 图标所表达的功能或物品 | "设置齿轮图标、背包图标、技能火焰图标" |
| 风格 | 与项目现有图标一致的美术风格 | "扁平化风格，圆角设计，2px描边" |
| 配色 | 主色和辅色，需与项目调色板一致 | "主色#FF6B35，辅色#FFFFFF，描边#333333" |
| 背景 | 图标底板或透明要求 | "透明背景"或"圆角方形底板，#2A2A2A" |

**风格一致性要求：**
- 每次生成图标时，**必须附带已有图标作为风格参考**
- 新生成的图标需通过风格一致性检查：描边粗细、圆角半径、光影方向、色彩饱和度均须与现有图标统一
- 若项目尚无图标风格规范，首次生成时需产出 3-5 个风格方案供选择，确定后固化为标准

**完整示例：**

```
技能图标——火焰爆发，一团向上喷涌的烈焰，中心为白热色向外过渡为橙红，
项目统一风格：扁平化卡通风格，粗描边(3px)，圆角造型，色块分明，
主色为火焰橙红(#FF4500)到金黄(#FFD700)渐变，高光为纯白，
透明背景，128×128像素，图标边缘留有8px安全边距，
与现有技能图标风格完全一致
```

### 四、特效帧提示词模板（5模块结构）

**基础结构：** `[特效类型描述], [帧数与节奏], [分辨率], [背景要求], [风格关键词]`

| 模块 | 说明 | 示例 |
|------|------|------|
| 特效类型描述 | 特效的视觉内容和运动方式 | "火焰爆炸从中心向外扩散，带有火星飞溅" |
| 帧数与节奏 | 总帧数及运动节奏描述 | "12帧，前4帧快速扩张，中间4帧维持，后4帧消散" |
| 分辨率 | 单帧尺寸 | "256×256像素"或"512×512像素" |
| 背景要求 | 通常为透明或黑色背景 | "透明背景（RGBA）"或"黑色背景（用于叠加混合）" |
| 风格关键词 | 特效的视觉风格 | "卡通风格特效，色块分明，边缘锐利" |

**序列帧命名格式：** `vfx_{特效名称}_{帧序号}.png`（如 `vfx_fireburst_001.png`），帧号三位数补零。

**完整示例：**

```
技能特效——冰霜新星，从中心爆发出一圈冰晶碎片向外扩散并逐渐消散，
共12帧序列动画，第1-3帧快速爆发扩张，第4-7帧冰晶碎裂扩散，第8-12帧碎片逐渐缩小消散，
单帧512×512像素，
透明背景（RGBA），
卡通风格特效，冰蓝色(#7EC8E3)到白色渐变，边缘锐利清晰，冰晶带有锐利棱角，
输出为PNG序列帧，帧命名格式：vfx_frostnova_001.png 至 vfx_frostnova_012.png
```

## 美术资源输出规格表

| 资源类型 | 分辨率 | 格式 | 色彩模式 | 命名规范 | 文件大小上限 | 备注 |
|----------|--------|------|----------|----------|-------------|------|
| 角色立绘（角色全身图） | 2048×2048 | PNG | RGBA（32位含透明通道） | `char_{id}_stand.png` | 8MB | 角色居中，四周留8%安全边距 |
| 角色战斗图（战斗用角色图） | 512×512 或 1024×1024 | PNG | RGBA | `char_{id}_battle.png` | 2MB | 小体型角色用512，大体型用1024 |
| 角色头像（对话/列表用） | 256×256 | PNG | RGBA | `char_{id}_avatar.png` | 512KB | 仅截取头部至胸部区域 |
| 场景背景（战斗/探索场景） | 3840×2160（4K）或 1920×1080 | PNG / JPG | RGB（24位） | `scene_{场景名}_bg.png` | 12MB（4K）/ 4MB（1080P） | 远景场景可用1080P，核心场景用4K |
| UI图标（功能图标） | 128×128 | PNG | RGBA | `icon_{功能名}.png` | 128KB | 图标内容区域不超出96×96，四周留安全边距 |
| UI按钮 | 256×64 或 320×80 | PNG | RGBA | `btn_{按钮名}.png` | 256KB | 需提供常态、按下、禁用三态 |
| 特效序列帧（VFX） | 256×256 或 512×512（单帧） | PNG | RGBA | `vfx_{特效名}_{帧号}.png` | 单帧256KB | 帧号三位数补零，如001、002 |
| 物品图标 | 128×128 | PNG | RGBA | `item_{物品id}.png` | 128KB | 物品主体居中，占画面70%-80% |
| 技能图标 | 256×256 | PNG | RGBA | `skill_{技能id}.png` | 256KB | 需提供圆角遮罩适配版本 |
| Logo / 标题 | 矢量或 4096×1024 以上 | SVG / PNG | RGBA | `logo_{项目名}.png` | 不限 | 需提供矢量源文件和多种尺寸栅格化版本 |

### 命名规范补充

- **{id}**：使用角色的数字ID或英文代号，全小写，单词间用下划线分隔
- **{场景名}**：使用场景的英文简称，全小写，如 `forest_01`、`dungeon_boss`
- **{功能名}**：使用 UI 功能的英文描述，如 `settings`、`inventory`、`quest`
- **{帧号}**：三位数字，从 `001` 开始递增
- 所有文件名仅允许使用小写字母、数字和下划线，禁止使用空格、中文、特殊字符

## Ban-list 机制（禁用元素管理）

Ban-list 用于记录并自动排除已确认不适合项目的美术元素。

### 管理规则

1. **新增流程**：当玩家反馈或团队评审确认某类美术元素效果不佳时，将该元素加入 Ban-list
2. **存储格式**：Ban-list 以条目形式记录，包含 `禁用元素描述`、`加入原因`、`加入日期`、`关联项目`
3. **应用方式**：每次生成提示词时，自动将 Ban-list 中的元素追加到负面提示词中
4. **定期审查**：每个迭代周期末审查 Ban-list，移除已不再适用的条目

### Ban-list 条目示例

```yaml
- element: "兵马俑风格角色设计"
  reason: "玩家反馈不佳，与项目整体风格不搭"
  date: "2025-01-15"
  project: "精灵物语"

- element: "过度写实的血腥效果"
  reason: "项目定位为全年龄段，需避免"
  date: "2025-02-01"
  project: "通用"
```

## 色彩管理

### 调色板层级

| 层级 | 用途 | 色彩数量 | 说明 |
|------|------|---------|------|
| 主色调（Primary Palette） | 品牌识别、核心界面、主角配色 | 3-5色 | 贯穿整个项目的核心色彩，出现在60%以上的画面中 |
| 辅助色调（Secondary Palette） | 次要界面、NPC、环境元素 | 5-8色 | 与主色调搭配使用，丰富画面层次，占比约30% |
| 点缀色（Accent Colors） | 高亮提示、稀有物品、特效闪光 | 2-3色 | 少量使用以吸引注意力，占比不超过10% |
| 功能色（Functional Colors） | 血条、蓝条、经验条、品质颜色 | 按功能定义 | 有明确语义的颜色，如红色=危险/HP、蓝色=MP、绿色=恢复 |

### 色彩使用规则

1. **禁止随意取色**：所有美术资源中的颜色必须从已定义的调色板中选取，禁止使用调色板之外的颜色（除非经美术主管审批新增）
2. **色值精确**：所有颜色使用 HEX 六位色值标注，禁止使用近似色或目测取色
3. **暗色模式适配**：如需支持暗色模式，每套调色板需提供亮色/暗色两个版本
4. **色盲友好**：功能色需考虑色盲用户，关键信息不可仅靠颜色区分，需辅以图标或形状标识
5. **色彩一致性检查**：每次产出新资源后，使用色彩检查工具验证所用颜色是否全部在调色板范围内

## 通用反馈机制

所有美术工作流均遵循统一的反馈循环机制，并包含以下美术特有的扩展：

- **美术ban位**：定期投喂玩家反馈不好的美术内容上ban位，后续产出直接pass此类风格
- **需人工确认标签**：需要美术组同学协同判断的情况，AI应标注"需人工确认"标签
- **风格一致性检查**：每次产出后检查与已有资产的风格一致性，不一致则打回
