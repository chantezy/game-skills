# 游戏剧情策划：AI辅助工作流

> **模块类型：类型池** | 适用：有叙事故事的游戏 | 不适用于：纯竞技/街机/无剧情游戏

## 工作流总览

剧情策划AI处理流程分为四个阶段，按顺序推进。每个阶段均有明确的输入、AI处理、产出和反馈机制。

```
策划初版大纲 → 步骤1: 结构优化与风险校验
    │
    ├── 打回 → 步骤2: 迭代修订 → 重新进入步骤1
    │
    ▼ 通过
细化文本 → 步骤3: 标准化处理与润色
    │
    ▼
定稿前文本 → 步骤4: 逻辑一致性校验
    │
    ├── 不一致 → 打回重新产出
    │
    ▼ 通过
终稿定稿
```

---

## 步骤1：剧情大纲AI结构优化与风险校验

### 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 策划初版剧情大纲 | 策划人员编写的原始剧情大纲 | 是 |
| 项目硬性要求 | 世界观约束、审核合规、IP限制等不可变要求 | 是 |
| 优化方向 | 用户期望的优化侧重（如节奏、角色弧线、冲突升级） | 否 |
| 过往评审意见 | 之前版本的评审反馈（如有） | 否 |

### AI处理

- 分析剧情大纲的结构合理性（起承转合、节奏分配、高潮排布）
- 主动发挥：基于游戏类型和世界观，提出结构优化方向
- 排查潜在风险（逻辑漏洞、合规风险、玩家接受度风险）
- 输出节奏排布方案（情绪曲线的起伏安排）

### 产出

| 产出项 | 说明 |
|--------|------|
| 优化建议清单 | 逐条列出结构问题及建议修改方向 |
| 风险排查报告 | 识别的逻辑漏洞、合规风险、敏感内容等 |
| 修订版大纲 | 基于优化建议修改后的大纲版本 |
| 节奏排布方案 | 剧情情绪曲线和各章节节奏规划 |

### 反馈机制

- 结构问题严重 → 打回进入步骤2进行迭代修订
- 无重大问题 → 进入剧情细化阶段（步骤3）

---

## 步骤2：打回大纲AI辅助迭代修订

> **注意：** 此步骤不一定每次都有，仅在大纲被打回时触发。

### 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 修改意见 | 评审方给出的具体修改意见 | 是 |
| 世界观/合规限制 | 不可违反的世界观设定和合规红线 | 是 |

### AI处理

- 分析打回原因，逐条拆解修改意见
- 反哺报告内容，将问题归类并产出针对性的提示词优化
- 主动发挥：基于修改意见联想可能的替代方案
- 确保修订后不再出现同类问题

### 产出

| 产出项 | 说明 |
|--------|------|
| 可落地方案清单 | 针对每条修改意见的具体落地方案 |
| 修订后大纲 | 整合修改意见后的新版大纲 |
| 同类问题规避提示 | 总结问题模式，给出后续创作中的规避建议 |

### 反馈机制

- 问题录入回风险记录，供后续阶段参考
- 修订后大纲需重新通过步骤1的校验
- 如果仍有问题 → 再次迭代，直到通过

---

## 步骤3：剧情细化文本AI标准化处理与润色

### 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 细化后的全量剧情文本 | 基于通过大纲展开的完整剧情文本 | 是 |
| 文案规范 | 项目的文案风格规范（语气、称谓、术语等） | 是 |
| 特殊优化需求 | 用户指定的特殊优化方向（如某角色台词风格、某场景氛围） | 否 |

### AI处理

- 标准化处理：统一术语、称谓、格式，确保符合文案规范
- 主动发挥：优化台词表现力，提升角色语言的个性化程度
- 润色文本：提升可读性，优化叙事节奏和情绪表达
- 校验一致性：检查前后文的角色行为、时间线、设定是否一致

### 产出

| 产出项 | 说明 |
|--------|------|
| 修正版文本 | 完成标准化处理后的剧情文本 |
| 润色优化台词 | 经过表现力提升的台词对照表（原文→优化后） |
| 可读性优化文本 | 叙事节奏和可读性优化后的完整文本 |
| 一致性校验报告 | 前后文一致性的检查结果 |

### 反馈机制

- 文本质量问题 → 优化提示词后重新生成
- 一致性发现问题 → 标记并打回相关段落

---

## 步骤4：剧情终稿AI逻辑一致性校验

### 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 定稿前全量文本 | 即将定稿的完整剧情文本 | 是 |
| 核心世界观规则 | 项目世界观的核心规则和不可违反的设定 | 是 |

### AI处理

- 全面检查时间线一致性（事件先后、角色年龄、季节变化等）
- 检查角色行为逻辑一致性（性格、动机、能力是否前后矛盾）
- 检查世界观设定一致性（魔法体系、政治格局、地理设定等）
- 主动发挥：交叉比对全量文本中的细节，发现潜在矛盾

### 产出

| 产出项 | 说明 |
|--------|------|
| 问题排查报告 | 逐条列出发现的不一致问题及其位置 |
| 修正建议方案 | 针对每个问题的具体修正建议 |

### 反馈机制

- **不一致的情况**：打回对应内容，重新产出优化的提示词，修正后再次校验
- **出问题的情况**：打回问题段落，重新产出优化的提示词，直到全部通过
- 全部通过 → 终稿定稿
