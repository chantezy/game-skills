# 游戏创意点讨论

## 概述

本技能定义游戏策划的**最前置环节——创意点讨论**。当用户输入一句话或一段话创意点时，AI不直接产出策划案，而是通过**引导式讨论**与用户探讨游戏性的实现，等讨论足够完善后，整理成大纲供用户确认，**再根据确认的大纲进入后续策划案生成流程**。

本环节只做讨论和大纲整理，不直接产出策划案。

## 核心理念

- **讨论优先于产出**：先充分讨论游戏性的实现，而非急于生成策划案
- **引导式而非索取式**：AI主动提出实现方向和问题，引导用户明确游戏性
- **大纲确认后才生成**：讨论充分后整理成大纲，用户确认后才进入策划案生成流程
- **用户提供优先**：参考内容、案例、素材等由用户提供；若用户未提供，AI基于游戏创意生成合适的内容

## 整体流程

```
用户创意点（一句话/一段话/关键词）
    │
    ▼
AI解析创意点 → 提取核心要素
    │
    ▼
引导式讨论阶段（核心环节）
    ├── 讨论游戏类型与核心机制实现
    ├── 讨论目标受众与平台适配
    ├── 讨论视觉风格与情绪体验
    ├── 讨论核心循环与玩法深度
    ├── 讨论风险与可行性
    └── （持续讨论直至充分完善）
    │
    ▼
讨论充分性判断
    ├── 不充分 → 继续讨论缺失维度
    └── 充分 → 进入大纲整理
    │
    ▼
整理为游戏设计大纲
    │
    ▼
用户确认大纲
    ├── 不确认 → 标注修改点 → 回到讨论
    └── 确认 → 根据大纲路由到各模块工作流生成策划案
```

## 创意点接收

### 最小输入

用户可以提供的最小输入：

- **一句话**：如"我想做一个用音乐控制植物生长的游戏"
- **一段话**：如"一个赛博朋克风格的侦探游戏，玩家通过破解数字线索破案，每个案件都有多个结局，玩家的选择会影响城市的社会治安值"
- **几个关键词**：如"太空 + 生存 + 建造 + 多人"

### 可选的用户提供内容

用户可主动提供以下内容，帮助AI更精准讨论（**均非必需，未提供则AI基于创意生成**）：

- 参考游戏或案例（如"参考《XX》的关卡设计"）
- 视觉风格参考图或描述
- 目标受众画像
- 平台约束
- 技术栈或引擎限制
- 美术素材或风格参考
- 世界观设定资料

### 创意点解析

AI接收到创意点后，立即进行解析，提取以下要素（缺失项进入讨论阶段补充）：

```
用户创意点
    │
    ▼
AI解析创意点，提取要素：
├── 游戏类型（动作/策略/RPG/模拟/解谜/...）
├── 核心机制（玩家主要在做什么）
├── 主题/题材（科幻/奇幻/现实/...）
├── 情绪体验（紧张/放松/探索/成就感/...）
├── 目标受众（可选，缺失则讨论中明确）
├── 平台（可选，缺失则讨论中明确）
└── 视觉风格（可选，缺失则讨论中明确）
    │
    ▼
进入引导式讨论
```

## 引导式讨论（核心环节）

### 讨论原则

AI遵循以下6条原则与用户讨论：

1. **每次只讨论1-2个维度**：不要一次性覆盖所有维度，避免信息过载
2. **不明确就给方向让用户选**：提供2-4个可选方向，而非开放提问
3. **给出默认建议**：基于已解析要素提供合理建议，供用户确认或调整
4. **确认而非询问**：用"我理解这个游戏是X风格，对吗？"代替"你想要什么风格？"
5. **聚焦实现可行性**：讨论重点是"如何实现"而非"是否要做"，关注游戏性的落地
6. **用户提供优先**：若用户提供了参考内容，AI基于用户内容展开；若未提供，AI基于游戏创意生成合适的方向供选择

### 讨论维度清单

按以下5个维度逐一讨论，每个维度讨论充分后再进入下一个：

#### 维度1：游戏类型与核心机制实现

**讨论重点：** 玩家在游戏中具体做什么，核心机制如何实现

**引导示例：**
```
AI：根据您的描述"用音乐控制植物生长的游戏"，我理解核心机制是"音乐驱动植物生长"。
请问核心玩法方向偏向哪个？
  A. 节奏游戏型——按节拍点击让植物按节奏生长
  B. 创造沙盒型——用不同旋律自由塑造植物形态
  C. 解谜关卡型——用音乐解开植物生长谜题过关
  D. 其他（请描述）

实现可行性分析：
- A方案实现难度：中（需音频节奏识别 + 生长动画同步）
- B方案实现难度：高（需旋律解析 + 自由形态生成系统）
- C方案实现难度：低（关卡化设计，机制可控）
您倾向哪个方向？
```

#### 维度2：目标受众与平台适配

**讨论重点：** 为谁做、在什么平台玩，影响系统深度和操作方式

**引导示例：**
```
AI：您选择了B. 创造沙盒型。这个方向很适合治愈系体验。
目标受众偏向？
  A. 休闲玩家（简单上手，短时游玩，单局5-10分钟）
  B. 创意玩家（深度创造，长时投入，单局30分钟+）
平台建议：
  A方案受众 → 移动端（触控操作，碎片化场景）
  B方案受众 → PC端（键鼠操作，深度体验）
您的选择？
```

#### 维度3：视觉风格与情绪体验

**讨论重点：** 视觉表现如何支撑情绪体验

**引导示例：**
```
AI：受众是休闲玩家，移动端平台。基于治愈系主题，推荐视觉风格：
  A. 水彩手绘风（温暖、柔和，适合治愈情绪）
  B. 低多边形Low Poly（清新、简约，适合放松）
  C. 像素风（复古、可爱，适合怀旧）
  D. 其他（请描述或提供参考图）

情绪体验目标：
- 您希望玩家主要感受到：[放松/成就感/探索欲/创造欲]？
```

#### 维度4：核心循环与玩法深度

**讨论重点：** 玩家在游戏中重复做什么，如何保持长期兴趣

**引导示例：**
```
AI：基于前面的讨论，核心循环可能是：
  播放音乐 → 植物生长 → 收集素材 → 解锁新旋律/植物 → 播放更复杂的音乐 → ...

玩法深度讨论：
- 短期目标（单局）：[玩家在5-10分钟内要达成什么？]
- 中期目标（章节）：[玩家在数小时内要解锁什么？]
- 长期目标（全游戏）：[玩家通关后的追求是什么？]
- 是否需要社交/分享功能？
```

#### 维度5：风险与可行性

**讨论重点：** 实现风险、设计风险、市场风险

**引导示例：**
```
AI：基于前面的讨论，识别出以下潜在风险：
- 技术风险：音乐节奏识别精度可能影响体验
- 设计风险：创造沙盒型容易缺乏目标感，玩家流失风险
- 内容风险：需要大量植物素材和旋律素材，内容生产压力大
- 市场风险：治愈系赛道竞争激烈，需差异化卖点

规避建议：
- 技术风险 → 初期采用预设节奏模板，降低识别复杂度
- 设计风险 → 加入"花园目标"任务系统，给玩家明确方向
- 内容风险 → 建立素材生成工作流，批量产出
- 市场风险 → 突出"音乐×植物×创造"的独特组合

您对哪些风险最关注？是否有补充？
```

## 讨论充分性判断

AI在每个维度讨论后，评估是否充分，判断标准：

| 维度 | 充分标准 | 不充分时处理 |
|------|---------|-------------|
| 游戏类型与核心机制 | 明确核心玩法方向，机制可实现 | 继续提供方向选项 |
| 目标受众与平台 | 明确受众画像与平台 | 提供受众+平台组合建议 |
| 视觉风格与情绪 | 明确风格与情绪目标 | 提供风格方向+情绪映射 |
| 核心循环与玩法深度 | 明确短/中/长期目标 | 引导用户思考各时间尺度 |
| 风险与可行性 | 识别主要风险并有规避思路 | 引导用户排序风险优先级 |

**整体讨论充分判断：** 五个维度均达到充分标准，且用户无新增补充，方可进入大纲整理。

## 快速启动模式

当用户在创意点中**一次性提供4项及以上信息**时（如同时给出游戏类型、核心机制、目标受众、平台、视觉风格中的多项），AI应跳过逐维度讨论，直接进入以下处理：

1. **快速解析**：提取用户已提供的所有要素，逐一确认
2. **补全缺失维度**：仅对未提及的维度进行引导讨论（提供2-4个选项）
3. **快速整合**：将已确认和补全的内容直接整理为游戏设计大纲
4. **确认大纲**：呈现大纲供用户确认或修改

**示例：**
用户说："我想做一个赛博朋克风格的RPG，PC端，面向硬核玩家，核心机制是黑客入侵+近战格斗双战斗系统。"

→ AI识别到已提供：游戏类型(RPG)、视觉风格(赛博朋克)、平台(PC)、受众(硬核玩家)、核心机制(黑客入侵+近战格斗)
→ 仅需补全：核心循环、风险与可行性
→ 快速进入大纲整理

## 用户提供内容处理原则

当用户提供或未提供参考内容时，AI的处理原则：

| 内容类型 | 用户提供 | 未提供 |
|---------|---------|--------|
| 参考游戏/案例 | 基于用户提供的案例讨论 | 基于游戏创意生成合适的方向供选择 |
| 视觉风格参考 | 基于用户参考图/描述展开 | 基于游戏类型和主题生成3个风格方向 |
| 目标受众画像 | 尊重用户受众定位 | 基于游戏类型推荐受众画像 |
| 平台约束 | 遵循用户平台选择 | 基于游戏类型和受众推荐平台 |
| 技术栈 | 遵循用户技术约束 | 基于游戏类型建议合适技术方案 |
| 世界观资料 | 基于用户设定展开 | 基于游戏主题生成世界观方向 |

**关键原则：** AI生成的内容仅作为讨论方向，必须经用户确认后才纳入大纲。

## 游戏设计大纲模板

讨论充分后，AI将讨论成果整理为**游戏设计大纲**，作为用户确认和后续策划案生成的依据：

```markdown
# [游戏名称] - 游戏设计大纲

## 创意点（用户原始描述）
[完整保留用户的创意描述]

## 游戏类型
[标签化类型描述]

## 核心机制
[一段话描述玩家主要在做什么，及实现方式]

## 主题与情绪
- 主题：[题材方向]
- 情绪体验：[玩家预期获得的感受]

## 目标受众
[玩家画像，含核心受众与次核心受众]

## 平台
[目标平台及操作方式]

## 视觉风格
[风格描述 + 用户提供的参考或AI基于创意生成的方向]

## 核心循环
[玩家在游戏中重复进行的主要行为循环]

## 游戏特色（卖点）
1. [卖点1]
2. [卖点2]
3. [卖点3]

## 玩法深度
- 短期目标：[单局目标]
- 中期目标：[章节目标]
- 长期目标：[全游戏目标]

## 风险与规避
- [风险1]：[规避方案]
- [风险2]：[规避方案]

## 模块拆分
基于以上大纲，需进一步设计的模块：
- [ ] 系统策划（核心——哪些系统）
- [ ] 关卡策划（类型——有关卡时选取）
- [ ] 战斗策划（类型——有战斗时选取）
- [ ] 剧情策划（类型——有故事时选取）
- [ ] 角色与单位设计（类型——有角色/单位时选取）
- [ ] 数值策划（核心——数值体系）
```

## 大纲确认与后续路由

### 用户确认大纲

AI整理大纲后，呈现给用户确认：

```
AI：基于我们的讨论，我整理了游戏设计大纲（见上方）。
请确认以下要点：
1. 核心机制方向是否准确？
2. 目标受众与平台是否匹配？
3. 视觉风格方向是否满意？
4. 核心循环是否完整？
5. 风险规避方案是否可行？

请回复：
- "确认" → 进入策划案生成流程
- "需要修改XX" → 标注修改点，回到讨论环节
```

### 确认后路由到模块生成

大纲确认后，AI根据游戏类型从**模块池**中自动选取所需模块：

**核心模块（默认启用）：**

| 模块 | 说明 |
|------|------|
| 系统策划 | 所有游戏均需要 |
| 数值策划 | 几乎所有游戏均需要 |
| 美术生产 | 所有游戏均需要 |
| QA测试 | 所有游戏均需要 |
| 程序开发 | 所有游戏均需要 |

**类型模块池（按游戏特征选取）：**

| 游戏特征 | 选取模块 |
|---------|---------|
| 有关卡/阶段/场景 | 关卡策划 |
| 有战斗内容 | 战斗策划 |
| 有叙事故事 | 剧情策划 |
| 有角色/单位/敌人 | 角色与单位设计 |
