# 关卡策划：AI策划案工作流

> **模块类型：类型池** | 适用：有关卡/阶段/场景的游戏 | 不适用于：纯沙盒/纯社交/无阶段游戏

## 工作流概览

关卡策划与系统策划结构一致，差异点如下。

### 输入差异

- 关卡关键词（替代系统关键词）
- 游戏类型
- 同类型关卡玩法参考（由用户提供，若未提供则AI基于游戏创意生成关卡方向）

### 处理差异

- AI拆解核心要素 → **关卡目标**（替代系统目标）
- 原型设计 → 关键模块、**玩家在关卡中动线**（替代玩家在系统中行为）
- 产出 **关卡策划案**（替代系统策划案）
- AI可主动建议：尝试喂入场景原型图来设计相关动线

### 其余流程

与系统策划一致：AI生成创新思路 → 用户选择/调整 → 提示词优化 → 质量评估 → 美术需求拆分 → 规则文档规范 → 策划案内容集合

---

## 三阶段产出模式

所有关卡策划按三阶段推进：

### 第一阶段：需求大纲

**目标：** 明确关卡关键词边界，拆解核心要素，产出关卡目标。

**产出模板：**

```markdown
# [关卡名称] - 需求大纲

## 设计目标
[一句话描述该关卡要解决的核心问题]

## 核心关键词
- 关键词1：[含义与边界说明]
- 关键词2：[含义与边界说明]

## 关卡目标
[明确该关卡要达成的具体目标]

## 参考方向
- [用户提供或AI基于游戏创意生成的参考方向]
```

### 第二阶段：细化方案

**目标：** 生成创新思路，设计关键模块与玩家动线，产出原型方案。

**产出模板：**

```markdown
# [关卡名称] - 细化方案

## 独特机制
[描述该关卡的核心创新点]

## 核心乐趣
[玩家在这个关卡中获得的核心体验]

## 潜在风险
- 风险1：[描述与规避方案]
- 风险2：[描述与规避方案]

## 关键模块
| 模块名称 | 功能描述 | 玩家行为 |
|---------|---------|---------|
| 模块1 | [精确描述] | [玩家行为] |
| 模块2 | [精确描述] | [玩家行为] |

## 原型设计
[关键模块布局、玩家动线、UI线框图描述]
```

### 第三阶段：完整策划案

**目标：** 整合产出完整关卡策划案，含美术需求拆分与规则文档。包含：完整关卡流程、地图规划、敌人配置、谜题设计、奖励分布、难度评分、检查点设置。

## 关卡策划设计知识库

### 一、节奏控制

关卡节奏是玩家体验的核心。AI 设计关卡时必须遵循以下节奏原则：

**张力曲线标准模型：**
```
安全区(起) → 小挑战(承) → 休息 → 中挑战(转) → 休息 → 大挑战/Boss(合) → 奖励
  2min        3min         1min     5min         1min      5-8min           1min
```

**节奏规则：**
- 单次紧张时长不超过 3-5 分钟，之后必须有喘息（安全区/奖励/剧情演出）
- Boss 战前必须有 1-2 分钟的安全准备区（补给/存档）
- 新机制引入时，先在安全环境中让玩家练习，再在压力环境中考验

**"四拍节奏"设计法（任天堂模式）：**

| 节拍 | 目的 | 示例（跳跃机制） |
|------|------|----------------|
| 起：引入 | 在安全环境中展示新机制 | 平地前方有个小坑，跳一下就过去 |
| 承：练习 | 低风险环境反复练习 | 连续几个坑，失败只是重走一小段 |
| 转：变体 | 增加难度或引入变体 | 移动平台上的坑、需要二段跳的宽坑 |
| 合：综合 | 综合考验已学技能 | Boss 战中需要在攻击间隙跳跃躲避 |

**四拍节奏的应用要点：**
- 每个新机制都必须经历完整的四拍循环
- "起"阶段绝对不能有死亡惩罚
- "转"阶段引入的变体应与前面学过的机制组合，而非全新机制
- "合"阶段是成就感高峰，应紧跟奖励

### 二、空间设计

| 设计维度 | 方案光谱 | 选择依据 |
|---------|---------|---------|
| 开放度 | 线性走廊 → 宽线性(分支小路) → 箱庭 → 开放世界 | 叙事驱动用线性，探索驱动用开放 |
| 引导技巧 | 光线引导/地标引导/敌人吸引/路径收窄/奖励引诱 | 开放世界至少用2种引导手段 |
| 垂直空间 | 跳跃平台/攀爬/飞行/电梯 | 增加空间层次感，但增加开发成本 |
| 视线管理 | 关键地标始终可见/刻意遮挡制造惊喜 | 开放世界需要远景地标 |

**空间设计决策流程：**
1. 先确定开放度（由游戏类型和叙事需求决定）
2. 根据开放度选择引导技巧数量（线性1种、宽线性2种、开放世界2-3种）
3. 评估是否需要垂直空间（预算充足且游戏类型需要时加入）
4. 最后设计视线管理（确保关键路径有视觉锚点）

### 三、难度梯度

| 阶段 | 关卡占比 | 难度特征 | 设计目标 |
|------|---------|---------|---------|
| 前段(教学) | 前 15-20% | 低难度，容错高，逐步引入机制 | 让玩家学会操作，建立信心 |
| 中段(发展) | 中间 50-60% | 中等难度，组合已学技能，引入变体 | 持续挑战，保持心流 |
| 后段(高潮) | 后 20-25% | 高难度，综合运用，允许试错 | 成就感巅峰，Boss 战 |

**难度参数可调整维度：**
- 敌人数量
- 敌人血量
- 敌人攻击频率/伤害
- 关卡时间限制
- 资源稀缺度
- 平台跳跃间距
- 谜题复杂度

**难度梯度设计原则：**
- 教学阶段：每引入1个新机制，给2-3次练习机会
- 发展阶段：组合2-3个已学机制，形成新挑战
- 高潮阶段：综合运用所有已学技能，允许失败重来
- 避免"难度断崖"——相邻区域的难度差不能超过一个台阶

### 四、关卡参数配置

每个关卡必须定义的参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| 目标完成时间 | 设计预期的通关时长 | 15-20 分钟 |
| 敌人配置 | 种类/数量/出现位置/巡逻路线 | 见下方配置示例 |
| 检查点间距 | 死亡后重来的最大回溯距离 | 不超过 3 分钟路程 |
| 奖励分布 | 宝箱/收集品的位置和内容 | 每 5 分钟至少一个小奖励 |
| 难度评分 | 1-5 星，基于上述参数综合评估 | ★★★☆☆ |

**敌人配置示例格式：**

| 区域 | 敌人类型 | 数量 | 位置 | 行为 |
|------|---------|------|------|------|
| 入口走廊 | 近战小兵 | 3 | 走廊中段 | 巡逻，发现玩家后追击 |
| 中央广场 | 远程弓箭手 | 2 | 高台 | 固定位置，射程15m |
| Boss房间前 | 精英守卫 | 1 | 门前 | 主动巡逻，召唤增援 |

**关卡参数校验清单：**
- [ ] 目标完成时间是否在合理范围（单关卡10-45分钟）
- [ ] 检查点间距是否不超过3分钟路程
- [ ] 每5分钟是否有至少一个小奖励
- [ ] 难度曲线是否平滑（无断崖式跳变）
- [ ] 新机制是否遵循四拍节奏引入
- [ ] Boss战前是否有安全准备区

## 与其他技能的协作

关卡策划案产出时,以下内容不在本技能范围内,由对应技能产出:

- **关卡内敌人的数值(HP/伤害/掉落)** → 由 `game-numerical-design` 基于本模块的敌人配置反推产出
- **关卡内战斗设计(Boss 招式/攻击手感)** → 由 `game-combat-design` 产出
- **关卡剧情触发与文本** → 由 `game-narrative-design` 产出
- **关卡配置表 Schema(关卡参数/敌人生成/掉落表)** → 由 `game-tech-implementation`(规范层)产出
- **关卡美术资源需求(场景/地标/特效)** → 由 `game-art-production` 产出

**前置依赖(建议先完成):**
- 系统策划案(确定核心玩法框架,关卡基于此设计)
