# 系统策划：AI策划案工作流

> **模块类型：核心** | 适用：几乎所有游戏

## 工作流概览

```
系统关键词 + 游戏类型（必需输入）
    │
    ├── 信息不足？──是──→ AI提供选项/默认方案 → 用户确认或补充
    │                 否
    │                 │
    ▼                 ▼
AI拆解核心要素 ──→ 系统目标
    │
    ▼
AI生成创新思路（主动发挥创造力）
    ├── 独特机制（AI主动提出多个方案）
    ├── 核心乐趣（AI联想玩家体验）
    └── 潜在风险（AI主动预判）
    │
    ▼
用户选择/调整方向
    │
    ▼
原型设计 ──→ 关键模块 + 玩家在系统中行为
    │
    ▼
质量评估 ──不符合──→ 优化提示词 → 重新生成
    │符合
    ▼
美术需求拆分 / 规则文档规范
    │
    ▼
系统策划案内容集合（按GDD格式输出）
```

---

## 输入要求

**必需：**
- 系统关键词（描述要设计什么系统）
- 游戏类型（决定系统设计方向）

**AI会主动引导补充：**
- 目标受众（影响系统复杂度和深度）
- 核心循环偏好（如用户未指定，AI会提供2-3个方向供选择）
- 参考方向（若用户未提供参考游戏/案例，AI基于游戏创意生成合适的设计方向供选择）

---

## AI处理流程

1. **需求澄清**
   - 角色定义：资深游戏系统设计师，拥有丰富游戏开发经验，擅长分析用户需求并将其转化为清晰的设计要素
   - 任务目标：分析输入的系统关键词和游戏类型，明确keyword的边界，避免歧义，转化为清晰的设计要素
   - AI主动发挥：基于游戏类型联想常见系统模式，提出创新方向

2. **AI拆解核心要素**：拆解出系统目标，AI主动提出可能的设计维度

3. **AI生成创新思路**：主动生成多个独特机制/核心乐趣方案，预判潜在风险

4. **原型设计**：设计关键模块、玩家在系统中行为，AI主动补全玩家行为路径

5. **质量评估**：符合需求则推进，不符合则优化提示词重新生成

6. **美术需求拆分**：AI根据策划案主动拆分视觉元素

7. **规则文档规范**：输出带规则注释的文档，确保程序员可直接编码

### 产出

- 系统目标
- 独特机制（多方案）
- 核心乐趣
- 潜在风险与规避方案
- 原型设计（关键模块、玩家行为）
- 美术需求拆分（视觉元素）
- 规则文档规范
- 带规则注释的文档（包含统计需求）
- 策划案内容集合

### 特殊说明

- 游戏图鉴和提示词固定的部分，可考虑批量产出或一键产出
- 对于产出准确的部分，可考虑一次指令自动分步产出
- AI生成测试用例
- 产品体验测试模块见QA部分

## 三阶段产出模式

所有系统策划按三阶段推进：

### 第一阶段：需求大纲

**目标：** 明确关键词边界，拆解核心要素，产出系统目标。

**产出模板：**

```markdown
# [系统名称] - 需求大纲

## 设计目标
[一句话描述该系统要解决的核心问题]

## 核心关键词
- 关键词1：[含义与边界说明]
- 关键词2：[含义与边界说明]

## 系统目标
[明确该系统要达成的具体目标]

## 参考方向
- [用户提供或AI基于游戏创意生成的参考方向]
```

### 第二阶段：细化方案

**目标：** 生成创新思路，设计关键模块与玩家行为，产出原型方案。

**产出模板：**

```markdown
# [系统名称] - 细化方案

## 独特机制
[描述该系统的核心创新点]

## 核心乐趣
[玩家在这个系统中获得的核心体验]

## 潜在风险
- 风险1：[描述与规避方案]
- 风险2：[描述与规避方案]

## 关键模块
| 模块名称 | 功能描述 | 玩家行为 |
|---------|---------|---------|
| 模块1 | [精确描述] | [玩家行为] |
| 模块2 | [精确描述] | [玩家行为] |

## 原型设计
[关键模块布局、玩家动线、UI线框图描述]
```

### 第三阶段：完整策划案

**目标：** 整合产出完整系统策划案，含美术需求拆分与规则文档。包含：完整的系统规则、配置表数据、数值参数、状态机定义、边界条件说明、美术资源清单。

## 系统策划设计知识库

AI 在设计系统时，应参照以下设计模式库，确保策划案具有实际深度而非停留在概念层面。

### 一、背包/物品系统

设计要点：

| 决策项 | 常见方案 | 选择依据 |
|-------|---------|---------|
| 容量模型 | 固定格位数 / 重量限制 / 无限+搜索 | 休闲游戏用无限+搜索，硬核游戏用格位/重量 |
| 堆叠规则 | 同类堆叠(上限99/999) / 不可堆叠 | 消耗品堆叠，装备/唯一物品不堆叠 |
| 排序方式 | 按类型/稀有度/获取时间/自定义 | 至少提供2种排序，默认按类型 |
| 快速操作 | 一键使用/一键穿戴/批量出售/自动整理 | 移动端必备快速操作 |
| 背包扩展 | 固定容量 / 付费扩展 / 任务解锁 | 付费点 or 进度奖励 |

**必须定义的参数：**
- 格位上限（默认40-60）
- 堆叠上限（消耗品99 / 材料999）
- 出售价格比（买入价的40-60%）
- 排序选项列表

### 二、技能/天赋系统

设计要点：

| 决策项 | 常见方案 | 选择依据 |
|-------|---------|---------|
| 结构模型 | 线性升级 / 树状分支 / 卡牌构筑 / 装备绑定 | 动作游戏用树状，Roguelike用卡牌/随机 |
| 技能点获取 | 升级获得 / 任务奖励 / 探索发现 | 多来源保证Build多样性 |
| 洗点机制 | 免费 / 付费消耗品 / 不可洗 | 免费洗点鼓励尝试，付费洗点是商业化点 |
| Build多样性 | 确保多条路线各有优势 | 避免唯一最优解，每个分支有适用场景 |
| 数值安全 | 技能叠加上限、百分比加算 vs 乘算 | 乘算防数值膨胀，加算直观易理解 |

**必须定义的参数：**
- 技能槽位数（通常3-5个主动 + 若干被动）
- 冷却区间（基础技能2-5s，大招15-60s）
- 资源消耗类型和数值
- 前置条件树

> **分工边界:** 系统策划只定义"技能系统的结构与成长机制",战斗表现(攻击手感参数、连招设计、Boss 招式、伤害公式)由 `game-combat-design` 产出。

### 三、经济系统

设计要点(**仅定义系统决策项,具体数值推导见 `game-numerical-design`**)：

| 决策项 | 常见方案 | 选择依据 |
|-------|---------|---------|
| 货币模型 | 单币 / 双币(软+硬) / 多币 | F2P必须双币,单机可单币 |
| 产出-消耗比 | 日产/日消 1.1-1.5 | <1.0 玩家流失,>2.0 通胀失控 |
| 价格锚定 | 以核心消耗品的"时间价格"为基准 | 一瓶回复药 = 3-5 分钟游戏时间 |
| 通胀控制 | 消耗坑(修理/税收/过期) + 产出封顶 | 长线游戏必须有消耗坑 |
| 交易体系 | 禁止交易 / 拍卖行 / 面对面 | 有交易就有经济管控需求 |

> **分工边界:** 系统策划只定义"经济系统要做哪些决策",数值推导(日产/日消比计算、时间价格锚定、30/60/90天资产曲线模拟)由 `game-numerical-design` 基于本模块的决策项反推产出。

### 四、任务系统

设计要点：

| 决策项 | 常见方案 | 选择依据 |
|-------|---------|---------|
| 任务类型 | 主线/支线/日常/周常/成就/隐藏 | 主线驱动剧情，日常维持DAU |
| 引导节奏 | 新手期密集→中期引导探索→后期自由 | 新手前30分钟任务密度最高 |
| 奖励节奏 | 即时(经验/金币) + 延迟(装备/称号) | 每次任务有即时反馈 |
| 任务追踪 | 自动追踪+导航 / 手动查看 | 开放世界需要导航辅助 |

### 五、存档系统

设计要点：

| 决策项 | 常见方案 | 选择依据 |
|-------|---------|---------|
| 存档点 | 手动存档 / 检查点自动存档 / 实时存档 | Roguelike无存档，RPG检查点+手动 |
| 槽位数 | 单槽 / 多槽(3-10) / 无限 | 单机RPG通常3-5个槽位 |
| 云同步 | 本地 / 云同步 / 强制云 | 跨平台必须云同步 |
| 防作弊 | 加密存档 / 服务端存档 / 校验和 | 竞技/排行榜游戏必须防篡改 |

### 六、社交系统

设计要点：

| 决策项 | 常见方案 | 选择依据 |
|-------|---------|---------|
| 社交类型 | 好友/公会/聊天/组队/交易/PvP | 按游戏类型选择需要的社交模块 |
| 匹配系统 | ELO/MMR / 随机 / 房间制 | 竞技游戏用ELO，休闲用随机 |
| 沟通工具 | 文字聊天/语音/快捷消息/表情 | 至少提供快捷消息+表情 |

---

## 与其他技能的协作

系统策划案产出时,以下技术对接要素不在本技能范围内,由对应技能产出:

- **配置表 Schema、状态机定义、网络同步策略、代码映射** → 由 `game-tech-implementation`(规范层)产出
- **经济系统的数值推导** → 由 `game-numerical-design` 基于本模块的决策项反推产出
- **技能系统的战斗表现** → 由 `game-combat-design` 产出
