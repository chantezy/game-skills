# 游戏数值策划：AI产出初始数值 + 平衡性分析

> **模块类型：核心** | 适用：几乎所有游戏（解谜/休闲可简化并入系统策划）

## 工作流概览

```
确定设计意图（打几轮？玩多久？花多少？）
    │
    ▼
AI推导初始数值（公式参数 + 配置表 + 推导过程）
    │
    ▼
平衡性模拟验证（极端Build测试 + 经济曲线模拟）
    │
    ├── 不通过 → 调整参数 → 重新验证
    │
    ▼ 通过
输出初始数值表 + 配置表 Schema + 平衡性报告
    │
    ▼
人工确认/微调 → 定稿
```

## 核心原则

**AI产出初始数值，附带推导过程，人工确认后定稿。**

数值是游戏的骨架，策划案必须包含可运行的初始数值。AI 的职责是：根据设计意图推导合理的初始数值，通过模拟验证平衡性，输出完整的配置表和推导说明。最终数值由人工确认。AI产出的是**真实数值+推导过程**，不是占位符或待定值。

## AI产出内容

1. **初始数值表**：所有系统的关键参数（战斗/经济/成长），附带推导过程说明
2. **配置表 Schema**：JSON 配置表结构定义
3. **平衡性分析报告**：极端Build测试、经济曲线模拟、30/60/90天资产预测
4. **活动资源总量分配情况**：根据活动规划产出资源分配方案
5. **对应执行代码**：根据数值需求产出执行代码（含注释，可审查）

## 数值推导方法论

### 一、战斗数值推导流程

#### 步骤1：确定设计意图

| 设计问题 | 通常范围 | 说明 |
|---------|---------|------|
| 普通怪打几轮？ | 3-5 轮 | 影响普通怪物HP |
| 精英怪打几轮？ | 8-12 轮 | 影响精英怪物HP |
| Boss 战持续多久？ | 2-5 分钟 | 影响Boss HP |
| 玩家能承受几次攻击？ | 3-7 次 | 影响玩家HP和怪物伤害 |

#### 步骤2：建立基础参数

```
设定基准值：
- 玩家初始攻击力: ATK_base = 100
- 每轮时间: T_round = 2.0s（包含攻击+等待）
- 普通怪目标轮数: N_normal = 4
```

#### 步骤3：反推公式参数

```
已知：
  伤害公式: DMG = ATK × skill_mult - DEF × def_reduction
  目标: 普通怪 HP 需要 N_normal 轮打死

推导：
  每轮伤害 = ATK_base × avg_skill_mult(假设1.2) - enemy_DEF × 0.5
           = 100 × 1.2 - enemy_DEF × 0.5
           = 120 - enemy_DEF × 0.5

  怪物HP = 每轮伤害 × N_normal
         = (120 - enemy_DEF × 0.5) × 4

  设定 enemy_DEF = 20:
  怪物HP = (120 - 10) × 4 = 440

验证：
  玩家DPS = 440 / (4 × 2.0) = 55/s
  BossHP(3分钟) = 55 × 180 = 9900
```

#### 步骤4：验证极端情况

| Build类型 | 攻击力 | 防御力 | 打普通怪轮数 | 打Boss轮数 | 被Boss打承受次数 | 结论 |
|----------|--------|--------|------------|----------|----------------|------|
| 均衡型 | 100 | 50 | 4 轮 | 3 min | 5 次 | 基准线 |
| 纯攻击 | 150 | 20 | 2.5 轮 | 2 min | 2 次 | 高风险高收益 |
| 纯防御 | 60 | 100 | 7 轮 | 5 min | 10 次 | 安全但缓慢 |

如果某个极端Build明显破坏平衡（如纯攻击1轮秒杀普通怪），需要调整公式参数。

### 二、经济系统数值推导

#### 步骤1：锚定核心消耗品的"时间价格"

```
设计决策：一瓶基础回复药 = 5 分钟游戏时间的产出
→ 回复药价格 = 5分钟产出量

如果玩家平均每分钟获得 20 金币:
→ 回复药价格 = 5 × 20 = 100 金币
```

#### 步骤2：以此锚定所有物品价格

| 物品 | 时间价格 | 金币价格 | 推导依据 |
|------|---------|---------|---------|
| 基础回复药 | 5 min | 100 | 锚定基准 |
| 中级回复药 | 15 min | 300 | 回复量3倍 |
| 基础武器 | 30 min | 600 | 约半小时游玩 |
| 稀有武器 | 120 min | 2400 | 约2小时游玩 |
| 传说武器 | 600 min | 12000 | 约10小时游玩 |

#### 步骤3：计算日产/日消比

```
假设每日游玩 1 小时:
- 日产出 = 60 min × 20 gold/min = 1200 gold
- 日消耗 = 药水(200) + 修理(100) + 其他(200) = 500 gold
- 日产/日消比 = 1200 / 500 = 2.4  ← 偏高！

调整方案：
- 增加消耗坑: 装备强化材料(300/day) + 技能升级(200/day)
- 调整后日消耗 = 1000 gold
- 调整后日产/日消比 = 1200 / 1000 = 1.2  ← 健康区间（目标 1.1-1.5）
```

#### 步骤4：模拟玩家资产曲线

| 时间点 | 累计产出 | 累计消耗 | 净资产 | 可购买的最好装备 | 设计意图 |
|--------|---------|---------|--------|----------------|---------|
| 7天 | 8400 | 5600 | 2800 | 中级武器 | 刚解锁核心系统 |
| 30天 | 36000 | 24000 | 12000 | 稀有武器 | 中期内容 |
| 60天 | 72000 | 48000 | 24000 | 史诗武器 | 后期内容 |
| 90天 | 108000 | 72000 | 36000 | 传说武器 | 终局内容 |

如果某时间段净资产增长过快或过慢，调整产出或消耗参数。

### 三、成长曲线推导

#### 步骤1：确定总游玩时长和升级节奏

```
设计目标：主线 20 小时通关，最大等级 50
升级节奏：
- 前期(1-10级)：每 15-30 分钟升一级 → 快速上手
- 中期(11-30级)：每 30-90 分钟升一级 → 稳定推进
- 后期(31-50级)：每 1-3 小时升一级 → 终局挑战
```

#### 步骤2：拟合经验曲线

```
经验公式: EXP_needed(level) = base × level^exponent

参数选择：
- base = 100, exponent = 1.8（适合中等节奏）
- base = 80, exponent = 2.0（适合前快后慢）
- base = 120, exponent = 1.5（适合较均匀的升级节奏）

以 base=100, exponent=1.8 为例：
  Lv1→2: 100 × 1^1.8 = 100 EXP (约5分钟)
  Lv10→11: 100 × 10^1.8 = 6310 EXP (约20分钟)
  Lv30→31: 100 × 30^1.8 = 41214 EXP (约1小时)
  Lv49→50: 100 × 49^1.8 = 103046 EXP (约2.5小时)

总经验 = Σ(level=1 to 49) EXP_needed(level) ≈ 1,565,000 EXP
对应游玩时长 ≈ 1,565,000 / (平均130 EXP/min) ≈ 200小时（含支线）
主线经验 ≈ 60% = 120小时 → 需要调整参数
```

#### 步骤3：属性成长曲线

```
每级属性提升 = 基础值 × (1 + growth_rate × level)

growth_rate 参考：
- HP: 0.08-0.12（每级增长 8-12%）
- ATK: 0.06-0.10
- DEF: 0.05-0.08
- SPD: 0.02-0.04（速度增长应缓慢，避免后期过快）
```

## 数值策划设计知识库

### 常见数值问题与解决方案

| 问题 | 症状 | 原因 | 解决方案 |
|------|------|------|---------|
| 数值膨胀 | 后期伤害是前期的1000倍+ | 乘算叠加无上限 | 改用加算，或设置叠加上限 |
| 通胀失控 | 玩家金币溢出，商店物品无意义 | 产出 > 消耗，缺乏消耗坑 | 增加消耗坑(强化/税收/过期) |
| 难度断崖 | 某关卡突然变难/变简单 | 数值曲线不连续 | 检查该关卡的敌人参数是否有跳变 |
| 一招鲜 | 某个技能/Build碾压一切 | 缺乏克制机制 | 增加属性克制/免疫/反制机制 |
| 无用选择 | 某个选项没人用 | 数值明显弱于其他 | 提升该选项的基础数值或独特机制 |

### 数值输出格式要求

AI 产出的每组数值必须包含以下五项：

1. **配置表数据**：JSON Schema 格式的配置表结构
2. **推导过程**：每个关键数值的设计意图和计算过程
3. **可调范围**：每个参数的上下限，超出后的影响说明
4. **关联影响**：修改此参数会影响哪些系统（便于后续调优）
5. **验证报告**：极端Build测试 + 经济曲线模拟的结果
