# 游戏角色与单位设计：AI辅助设计流程

> **模块类型：类型池** | 适用：有可控角色/敌方单位/NPC的游戏 | 不适用于：抽象/纯益智游戏

## 概述

本模块覆盖游戏中所有**可控角色、敌方单位、NPC、载具、召唤物**等"游戏内实体"的设计。不同游戏类型的角色设计差异很大——RPG 设计的是英雄和 NPC，策略游戏设计的是兵种，收集养成类设计的是精灵/宠物，FPS 设计的是敌方 AI。本模块用统一的五阶段流程适配所有类型。

## 适用性判断

| 游戏类型 | 是否启用 | 设计重点 |
|---------|---------|---------|
| RPG/ARPG | 必须 | 主角、队友、NPC、敌人、Boss |
| 策略/战棋 | 必须 | 兵种、英雄、敌方单位 |
| 收集养成（宝可梦类） | 必须 | 精灵/宠物，含进化链和变体 |
| 动作/格斗 | 必须 | 可操作角色、Boss、小怪 |
| FPS/TPS | 必须 | 敌方AI类型、队友、Boss |
| MOBA | 必须 | 英雄角色池、小兵、野怪 |
| 模拟经营 | 可选 | 员工/居民（如有角色系统） |
| 解谜/休闲 | 通常不需要 | 仅在有引导NPC时启用 |
| 赛车/体育 | 可选 | 载具设计（替代角色） |

## 步骤1：原型调研与定位分析

### 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 角色核心定位 | 在游戏中的功能和角色（如"火属性近战输出"） | 是 |
| 游戏类型与世界观约束 | 决定调研方向和原型范围 | 是 |
| 调研维度要求 | 用户指定的调研侧重点 | 否 |

### AI处理（按游戏类型分支）

| 游戏类型 | 调研方向 | 产出重点 |
|---------|---------|---------|
| RPG/ARPG | 职业原型（战士/法师/游侠等）、神话/历史人物参考 | 职业定位、角色弧线方向 |
| 策略/战棋 | 历史兵种、军事编制、克制关系 | 兵种功能、克制链设计 |
| 收集养成 | 现实生物原型、民俗文化、生态位分析 | 种族分类、属性分配、进化方向 |
| 动作/格斗 | 格斗流派、武术风格、武器类型 | 战斗风格、招式方向 |
| FPS/TPS | 军事单位职能、装备配置 | 敌方行为模式、战术定位 |
| MOBA | 角色原型、能力类型（刺客/坦克/辅助） | 角色定位、技能方向 |

### 通用产出

- 原型调研报告（现实/文化/游戏参考）
- 设计建议清单
- 风险规避提示（同质化风险、文化敏感风险）
- 标杆拆解（同类型优秀游戏的角色设计分析）

## 步骤2：设定生成与标准化

### 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 核心需求单 | 角色的功能需求描述 | 是 |
| 标准化模板 | 根据游戏类型选择对应模板 | 是 |
| 同生态位/同定位参考 | 同类型角色的参考设定 | 否 |
| 步骤1产物 | 原型调研报告和设计建议 | 是 |

### AI处理：按游戏类型选择对应设定模板

**RPG/ARPG 角色设定模板：**

```
- 名称/代号
- 种族/出身
- 职业定位（坦克/输出/辅助/混合）
- 背景故事（100字以内）
- 性格特征（影响对话和行为）
- 核心属性档位：HP / ATK / DEF / SPD / LUK（高/中/低）
- 技能方向：2-3个标志性技能的概念
- 成长弧线：角色在剧情中的变化
```

**策略/战棋 兵种设定模板：**

```
- 兵种名称
- 兵种分类（步兵/骑兵/弓兵/法师/攻城/...）
- 移动力/攻击范围/移动方式
- 核心属性：HP / ATK / DEF / 移动力
- 克制关系：克制谁、被谁克制
- 特殊能力：1-2个独特能力
- 升级路线：可转职为哪些高级兵种
```

**收集养成 精灵/宠物设定模板：**

```
- 精灵名称/编号
- 种族/属性（火/冰/雷/风/地/...，按游戏世界观定义）
- 稀有度档位（普通/稀有/史诗/传说）
- 核心定位（攻击/防御/速度/辅助）
- 进化链：幼体 → 成体 → 终极体（各阶段特征变化）
- 特性/被动能力：1-2个
- 数值档位：HP / ATK / DEF / SPD（S/A/B/C/D 五档）
- 生态设定：栖息地、习性、食物链关系
```

**FPS/TPS 敌方单位设定模板：**

```
- 单位名称
- 单位类型（步兵/重装/狙击/特殊/精英/Boss）
- 行为模式：巡逻/掩护/冲锋/迂回/撤退的条件
- 武器装备：主武器/副武器/投掷物
- 核心属性：HP / 精准度 / 反应速度 / 护甲
- 战术角色：前线/火力支援/侧翼包抄/指挥官
- 出现场景：在哪些关卡/情境下出现
```

### 通用产出

- 标准化设定文案（按上述对应模板）
- 数值档位建议
- 与已有角色的差异化分析（避免同质化）

### 反馈

- 没有问题 → 导入下一步（美术方向）
- 有风险 → 重新产出优化提示词

## 步骤3：美术方向与视觉设计

### 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 正式需求单 | 基于步骤2的设定产出 | 是 |
| 美术规范 | 整体风格、配色要求 | 是 |
| 产出要求 | 立绘/模型参考/头像/战斗图 | 否 |
| 禁用元素 | 明确不可出现的视觉元素 | 否 |

### AI处理

- 生成标准化AI绘画提示词
- 辅助产出多方案草图/色稿方向
- 检查与已有角色的视觉差异化（剪影辨识度、配色区分度）

### 产出

- 标准化AI绘画提示词
- 多方案草图/动态稿
- 配色方案色稿
- 视觉差异化检查报告

### 风险机制

- 定期投喂玩家反馈不好的美术内容上ban位，后续产出直接pass此类风格
- 需要美术组同学协同判断
- 评审打回内容记录回历史评审记录

## 步骤4：评审打回与优化调整

> **注意：** 仅在评审打回时触发，不是必经步骤。

### 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 评审打回的原版设计 | 设定文案或草图 | 是 |
| 修改意见 | 评审方的具体意见 | 是 |
| 原需求单 | 原始设计需求 | 是 |

### AI处理

- 分析打回原因，区分为三类：
  - **设定问题**：角色定位不清、功能重叠
  - **视觉问题**：风格不统一、辨识度低
  - **平衡问题**：数值过强/过弱
- 生成规避同类问题的调整方案
- 调整后重新产出

### 产出

- 优化方案清单
- 调整后设定/提示词/草图
- 同类问题规避提示

## 步骤5：变体与扩展设计

### 按游戏类型的变体方式

| 游戏类型 | 变体形式 | 示例 |
|---------|---------|------|
| 收集养成 | 异种/地区形态/闪光版 | 火属性狐狸 → 冰属性狐狸异种 |
| RPG | 转职/觉醒/皮肤 | 战士 → 骑士 / 狂战士 |
| 策略/战棋 | 高级兵种/精英变体 | 弓箭手 → 游侠 / 狙击手 |
| FPS/TPS | 精英版/变体AI | 普通步兵 → 重装步兵 / 老兵 |
| MOBA | 皮肤（纯外观）/ 新英雄 | 基于同一角色的不同皮肤 |
| 动作/格斗 | DLC角色/隐藏角色 | 基于已有招式框架的新角色 |

### 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 已定稿的角色/单位 | 作为变体设计的基础 | 是 |
| 变体设计需求 | 需要什么类型的变体 | 是 |
| 设计规范 | 变体需遵循的约束 | 否 |

### AI处理

- 基于定稿版本发散差异化变体方案
- 确保变体与原版有足够辨识度，同时保持设计语言统一

### 产出

- 多套变体方案
- 差异化亮点说明
- 优化后提示词
- 变体设计初稿

### 反馈

- 不符合需求 → 打回优化提示词

## 角色与单位设计知识库

### 角色差异化原则（"三不同"）

同一游戏中的角色/单位必须做到"三不同"：

| 维度 | 要求 | 检验方法 |
|------|------|---------|
| 功能不同 | 每个角色在战斗/玩法中有独特定位 | 移除该角色后，玩法体验有明显缺失 |
| 外观不同 | 剪影可辨识，配色不重复 | 涂黑后仍能区分各角色 |
| 体验不同 | 操作该角色时的感受不同 | 玩家切换角色后能感受到差异 |

### 角色数量规划

| 游戏类型 | 初始角色数 | 扩展目标 | 设计原则 |
|---------|----------|---------|---------|
| 动作/格斗 | 6-10 | 15-20 | 少而精，每个都有独特手感 |
| RPG | 4-6 可控 + 20+ NPC | 8-12 可控 | 初始覆盖基础职业 |
| 策略/战棋 | 8-12 兵种 | 20-30 | 初始覆盖基础克制链 |
| 收集养成 | 20-50 精灵 | 100+ | 初始覆盖全属性，稀有度分布合理 |
| MOBA | 10-15 英雄 | 30-50 | 初始覆盖全定位（坦/输出/辅/刺） |
| FPS/TPS | 5-8 敌方类型 | 15-20 | 初始覆盖基础战术角色 |

### 平衡性设计要点

- **克制环**：至少建立一组三角克制（A克B、B克C、C克A），避免单一最优解
- **对称与非对称**：PvP 游戏需要对称平衡（双方机会均等）或有意设计的非对称平衡（各有优势）
- **使用率监控**：设计时预判每个角色的使用率，目标是无人使用率 < 5%
- **Tier 控制**：避免设计"必选"角色——每个角色至少有一个counter（克制手段）

## 与其他技能的协作

角色与单位设计案产出时,以下内容不在本技能范围内,由对应技能产出:

- **角色战斗表现(攻击手感/连招/Boss 招式)** → 由 `game-combat-design` 产出
- **角色数值(HP/ATK/DEF/成长曲线)** → 由 `game-numerical-design` 基于本模块的角色定位与属性初始值反推产出
- **角色美术资源(立绘/模型/动作)** → 由 `game-art-production` 产出(基于本模块的外观描述)
- **角色配置表 Schema(character_defs/属性表)** → 由 `game-tech-implementation`(规范层)产出

**前置依赖(建议先完成):**
- 系统策划案(角色机制需符合系统框架)

**下游协作:**
- 战斗策划基于本模块的角色清单设计技能
- 剧情策划基于本模块的角色设定编写对白
- 数值策划基于本模块的属性初始值推导成长曲线
